# 砂石设备监测数据定时同步功能

## 功能说明

该功能每15分钟自动调用外部接口获取砂石设备监测数据，并将数据存储到数据库中。

## 文件清单

### 1. 实体类
- `SandEquipmentMonitor.java` - 监测数据实体类
- `EnvironmentMonitorNewVo.java` - 接口响应VO类

### 2. Mapper
- `SandEquipmentMonitorMapper.java` - MyBatis Plus Mapper接口

### 3. Service
- `SandEquipmentMonitorService.java` - 业务逻辑服务类
  - `fetchMonitorData()` - 调用外部接口获取数据
  - `saveMonitorData()` - 保存数据到数据库
  - `syncMonitorData()` - 完整的同步流程

### 4. Scheduler
- `SandEquipmentMonitorScheduler.java` - 定时任务调度器
  - 每15分钟执行一次（cron: `0 0/15 * * * ?`）

### 5. 数据库脚本
- `SQL/sand_equipment_monitor.sql` - 建表SQL

## 配置说明

### 1. 修改API地址

编辑 `src/main/resources/application-dev.yml`：

```yaml
environment:
  api:
    base-url: http://your-api-server:port  # 修改为实际的API地址
```

### 2. 执行建表SQL

在MySQL中执行 `SQL/sand_equipment_monitor.sql` 创建数据表。

## 定时任务配置

### Cron表达式说明

当前配置：`0 0/15 * * * ?`

含义：每15分钟的第0秒执行一次

| 位置 | 含义 | 值 |
|------|------|-----|
| 1 | 秒 | 0 |
| 2 | 分 | 0/15 (从0开始，每15分钟) |
| 3 | 时 | * (每小时) |
| 4 | 日 | * (每日) |
| 5 | 月 | * (每月) |
| 6 | 周 | ? (不指定) |

### 常用Cron表达式示例

```
每5分钟:   0 0/5 * * * ?
每10分钟:  0 0/10 * * * ?
每30分钟:  0 0/30 * * * ?
每小时:    0 0 0/1 * * ?
每天凌晨:  0 0 0 * * ?
```

## 注意事项

1. **确保Spring定时任务已启用**
   - 检查 `DatacenterApplication.java` 是否有 `@EnableScheduling` 注解

2. **网络连接**
   - 确保应用服务器能够访问外部API地址
   - 超时时间设置为30秒，可根据实际情况调整

3. **异常处理**
   - 定时任务有try-catch保护，单次失败不会影响下次执行
   - 所有异常都会记录到日志中

4. **数据去重**
   - 如果接口返回的数据包含ID，系统会判断是更新还是插入
   - 如果没有ID，则每次都插入新记录

5. **性能考虑**
   - 如果数据量很大，可以考虑批量插入优化
   - 可以添加数据清理策略，定期删除旧数据

## 常见问题

### Q1: 定时任务没有执行？

A: 检查以下几点：
1. 确认 `@EnableScheduling` 注解存在
2. 确认 `SandEquipmentMonitorScheduler` 类上有 `@Component` 注解
3. 查看应用启动日志，确认定时任务已注册

### Q2: 接口调用失败？

A: 检查：
1. API地址配置是否正确
2. 网络是否通畅
3. 查看日志中的错误信息

### Q3: 数据没有保存？

A: 检查：
1. 数据库连接是否正常
2. 表是否已创建
3. 查看日志中的SQL执行情况
