# 砂石设备监测数据查询接口文档

## 📋 概述

本接口用于获取砂石设备监测数据，对应前端页面展示需求：
- **左侧区域**：显示5类设备的最新实时数据（不受筛选条件影响）
- **右侧区域**：根据设备选择和日期范围展示历史数据趋势图（用于图表展示）

---

## 🔗 接口信息

### 基本信息

| 项目 | 内容 |
|------|------|
| 接口名称 | 获取砂石设备监测数据 |
| 请求地址 | `/api/sand-monitor/data` |
| 请求方式 | `GET` |
| Content-Type | `application/json` |

---

## 📥 请求参数

### Query 参数

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| deviceId | Integer | 否 | 设备ID（不传则查询所有设备的历史数据） | 2001 |
| startDate | String | 否 | 开始日期（格式：yyyy-MM-dd） | 2026-06-01 |
| endDate | String | 否 | 结束日期（格式：yyyy-MM-dd） | 2026-06-12 |

### 参数说明

1. **不传任何参数**：返回所有设备的最新数据 + 所有设备的所有历史数据
2. **只传 deviceId**：返回所有设备的最新数据 + 指定设备的所有历史数据
3. **只传日期范围**：返回所有设备的最新数据 + 所有设备在日期范围内的历史数据
4. **同时传 deviceId 和日期**：返回所有设备的最新数据 + 指定设备在日期范围内的历史数据

> **注意**：左侧的"最新数据"始终返回5条（每个设备一条），不受筛选条件影响。

##  使用场景

### 场景1：页面初始化加载

**请求**：
```
GET /api/sand-monitor/data
```

**用途**：
- 左侧显示5个设备的最新数据
- 右侧默认显示所有设备的所有历史数据趋势

### 场景2：选择特定设备查看趋势

**请求**：
```
GET /api/sand-monitor/data?deviceId=2001
```

**用途**：
- 左侧仍显示5个设备的最新数据（不变）
- 右侧只显示设备2001（破碎机）的历史数据趋势

### 场景3：查看指定时间段的数据

**请求**：
```
GET /api/sand-monitor/data?startDate=2026-06-10&endDate=2026-06-12
```

**用途**：
- 左侧仍显示5个设备的最新数据（不变）
- 右侧显示所有设备在6月10日-12日的历史数据趋势

### 场景4：组合筛选

**请求**：
```
GET /api/sand-monitor/data?deviceId=2000&startDate=2026-06-01&endDate=2026-06-12
```

**用途**：
- 左侧仍显示5个设备的最新数据（不变）
- 右侧只显示设备2000（采石场）在6月1日-12日的历史数据趋势

### 调用示例（JavaScript）

```javascript
// 1. 页面初始化：加载所有数据
async function loadInitialData() {
  const response = await fetch('/api/sand-monitor/data');
  const data = await response.json();
  
  // 渲染左侧：5个设备的最新数据
  renderLatestDevices(data.latestDataList);
  
  // 渲染右侧：所有历史数据的图表
  renderCharts(data.historyDataList);
}

// 2. 用户选择设备后：更新图表
async function onDeviceSelect(deviceId) {
  const params = new URLSearchParams({
    deviceId: deviceId
  });
  
  const response = await fetch(`/api/sand-monitor/data?${params}`);
  const data = await response.json();
  
  // 左侧保持不变（仍然是5个设备的最新数据）
  // 只更新右侧图表
  renderCharts(data.historyDataList);
}

// 3. 用户选择日期范围后：更新图表
async function onDateRangeChange(startDate, endDate) {
  const params = new URLSearchParams({
    startDate: startDate,
    endDate: endDate
  });
  
  const response = await fetch(`/api/sand-monitor/data?${params}`);
  const data = await response.json();
  
  // 左侧保持不变
  // 只更新右侧图表
  renderCharts(data.historyDataList);
}

// 4. 同时选择设备和日期
async function onFilterChange(deviceId, startDate, endDate) {
  const params = new URLSearchParams();
  
  if (deviceId) params.append('deviceId', deviceId);
  if (startDate) params.append('startDate', startDate);
  if (endDate) params.append('endDate', endDate);
  
  const response = await fetch(`/api/sand-monitor/data?${params}`);
  const data = await response.json();
  
  // 左侧保持不变
  // 只更新右侧图表
  renderCharts(data.historyDataList);
}

// 渲染图表函数（示例）
function renderCharts(historyDataList) {
  // 提取时间轴
  const timeLabels = historyDataList.map(item => 
    item.syncTime.substring(11, 16) // 提取 HH:mm
  );
  
  // 提取各指标数据
  const pm2Data = historyDataList.map(item => item.pm2);
  const pm10Data = historyDataList.map(item => item.pm10);
  const tempData = historyDataList.map(item => item.temperature);
  const humidityData = historyDataList.map(item => item.humidity);
  const noiseData = historyDataList.map(item => item.noise);
  const windSpeedData = historyDataList.map(item => item.windSpeed);
  
  // 使用 ECharts 或其他图表库渲染
  renderTemperatureChart(timeLabels, tempData, humidityData);
  renderParticleChart(timeLabels, pm2Data, pm10Data);
  renderNoiseChart(timeLabels, noiseData);
  renderWindSpeedChart(timeLabels, windSpeedData);
  // ... 其他图表
}
```

## 🔧 技术实现要点

### 1. 最新数据查询逻辑

```sql
-- 使用子查询获取每个设备的最新记录
SELECT m.* FROM sand_equipment_monitor m 
WHERE sync_time = (
  SELECT MAX(sync_time) FROM sand_equipment_monitor m2 
  WHERE m2.device_id = m.device_id
)
ORDER BY m.device_id
```

**特点**：
- ✅ 始终返回5条数据（每个设备一条）
- ✅ 不受筛选条件影响
- ✅ 按设备ID排序

### 2. 历史数据查询逻辑

```java
// 构建动态查询条件
LambdaQueryWrapper<SandEquipmentMonitor> wrapper = new LambdaQueryWrapper<>();

// 设备筛选
if (deviceId != null) {
    wrapper.eq(SandEquipmentMonitor::getDeviceId, deviceId);
}

// 日期筛选
if (startDate != null) {
    wrapper.ge(SandEquipmentMonitor::getSyncTime, startDate.atStartOfDay());
}

if (endDate != null) {
    wrapper.le(SandEquipmentMonitor::getSyncTime, endDate.atTime(LocalTime.MAX));
}

// 按时间正序排列（图表需要）
wrapper.orderByAsc(SandEquipmentMonitor::getSyncTime);
```

**特点**：
- ✅ 支持灵活的条件组合
- ✅ 按时间正序排列（适合图表展示）
- ✅ 无分页限制，返回所有符合条件的数据

### 3. 性能优化建议

1. **添加索引**：
   ```sql
   CREATE INDEX idx_device_sync ON sand_equipment_monitor(device_id, sync_time);
   ```

2. **缓存最新数据**：
   - 左侧的最新数据可以缓存30秒
   - 减少数据库查询压力

3. **定期清理历史数据**：
   - 保留最近3个月的数据
   - 避免数据量过大影响查询性能

## 📝 注意事项

1. **左侧数据固定**：无论怎么筛选，左侧始终显示5个设备的最新数据
2. **右侧数据可变**：右侧图表数据会根据筛选条件动态变化
3. **时间顺序**：历史数据按时间正序排列，方便图表从左到右展示
4. **Mock模式**：本地开发时可以通过 `environment.api.mock-enabled=true` 启用Mock数据
5. **数据量控制**：如果历史数据过多，建议在前端做数据采样或后端添加限制

## ❓ 常见问题

### Q1: 为什么左侧数据不受筛选条件影响？
A: 因为左侧是"最新监测数据"，应该始终显示当前所有设备的最新状态，与历史数据筛选无关。

### Q2: 历史数据太多怎么办？
A: 
- 方案1：前端做数据采样（如每隔5分钟取一个点）
- 方案2：后端添加limit限制（如最多返回1000条）
- 方案3：增加更细粒度的时间筛选

### Q3: 如何获取设备列表用于下拉框？
A: 可以从 `latestDataList` 中提取设备信息，或者单独提供一个设备列表接口。

### Q4: Mock数据如何关闭？
A: 修改配置文件 `application-dev.yml`：
```yaml
environment:
  api:
    mock-enabled: false  # 改为false即可
```
